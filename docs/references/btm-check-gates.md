# BTM Check Gates

Every gate runs on `pnpm run check`. Each supports `--explain` (why a match fires) and `--json` (machine-readable output).

| Gate                 | Script                                   | Purpose                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                          |
| -------------------- | ---------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Version consistency  | `scripts/check-version-consistency.mts`  | Cross-references `.gitmodules` version comments against each upstream's `package.json` `sources.<upstream>.version` + `.ref` and the actual gitlink SHA. Catches submodule bumps that forgot to touch the version table, or a version table pointing at a commit the submodule isn't on.                                                                                                                                                                                                                                                                                                         |
| Mirror-docs sync     | `scripts/check-mirror-docs.mts`          | Every public `lib/smol-*.js` module has a matching `docs/additions/lib/<name>.js.md`, and every mirror doc still has a live source. Catches orphaned docs and undocumented new public modules.                                                                                                                                                                                                                                                                                                                                                                                                   |
| Regression patterns  | `scripts/check-regression-patterns.mts`  | Encodes recurring bug shapes (regex patterns) caught across R14+ quality-scan rounds. Strict — no allowlist; fix the code (apply the `fix:` field remediation) so the regex no longer fires. To add a new pattern, edit `REGRESSIONS`, fix existing matches in the same commit, document the lesson in the commit message. The gate is regression-prevention only; `/quality-scan` still runs periodically to find NEW patterns.                                                                                                                                                                 |
| Cascade completeness | `scripts/check-cascade-completeness.mts` | Walks every Makefile `include`, cross-package TypeScript `import`, and Dockerfile `COPY` and verifies each dependency is covered by a CASCADE_RULE in `scripts/validate-cache-versions.mts` OR by a hash in the consuming workflow's cache-key composition. Allowlist non-build-affecting deps in `.github/cascade-completeness-allowlist.yml`.                                                                                                                                                                                                                                                  |
| Patch format         | `scripts/check-patch-format.mts`         | Validates every `.patch` under `packages/*/patches/`. Rules: `# @<project>-versions: vX.Y.Z` header on first non-blank line (project tag must match patch tree — node/lief/opentui); `# @description:` header present and non-empty; standard unified diff (`--- a/`, `+++ b/`), NOT `git format-patch` preamble; hunk header counts (`@@ -A,B +C,D @@`) match actual body line counts (blank-line tolerance matches `git apply`); one file per patch (both axes — within a patch AND across the series, each source file owned by exactly one patch); no gaps in numbered-series filenames. |

Related rule: `.claude/rules/gitmodules-version-comments.md` — `.gitmodules` version-comment format.

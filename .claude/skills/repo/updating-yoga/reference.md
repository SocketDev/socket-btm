## Version Reference

- Version pinned in: `packages/yoga-layout-builder/package.json` under `sources.yoga`
- Also tracked as submodule in .gitmodules: `# yoga-X.Y.Z` (e.g., `# yoga-3.2.1`)
- Submodule path: `packages/yoga-layout-builder/upstream/yoga`
- Upstream: `https://github.com/facebook/yoga.git`
- Tags format: `vX.Y.Z` (e.g., `v3.2.1`)
- The `ref` field in package.json is a commit SHA
- Exclude: Any tag with `rc`, `alpha`, `beta`

## Cache Version Dependencies

When updating Yoga, bump this cache version:

```json
{
  "versions": {
    "yoga-layout": "v21" // ← Bump this
  }
}
```

**Note:** Downstream consumers should be rebuilt to pick up the new `yoga-sync.mjs`.

## Wrapper Files Sync

`src/wrapper/` has two files with different upkeep:

| Local File | Upstream | Upkeep on a yoga bump |
|-----------|----------|------------------------|
| `src/wrapper/YGEnums.mts` | `yoga/YGEnums.h` | GENERATED by the build — never hand-edit, never diff |
| `src/wrapper/wrapAssembly.mts` | `yoga/javascript/src/wrapAssembly.ts` | Manual 1:1 re-port when upstream changed |

### YGEnums.mts is generated — leave it alone

`scripts/source-cloned/shared/generate-enums.mts` (run from `clone-source.mts`)
parses `YG_ENUM_DECL` blocks out of `upstream/yoga/yoga/YGEnums.h` and writes
`YGEnums.mts` on every build. The header and the `embind.cpp` the WASM
compiles from are the same pinned submodule checkout, so the enum values
can never drift from the binary's ABI. A yoga bump regenerates the file
automatically and re-stamps its `@ yoga <ver>` header. **Do not diff it,
do not hand-edit it** — just commit whatever the build regenerated.

### wrapAssembly.mts re-port

Diff upstream `wrapAssembly.ts` between the OLD and NEW yoga tags:

```bash
OLD=<previous ref>  NEW=<new ref>
git -C packages/yoga-layout-builder/upstream/yoga \
  diff "$OLD" "$NEW" -- javascript/src/wrapAssembly.ts
```

- **No diff** → nothing to do; the build re-stamps the version marker.
- **Diff present** → re-port the behavioral change into our
  `src/wrapper/wrapAssembly.mts` using the fleet-canonical locked-down
  agent (NOT a free-form `claude` call — per CLAUDE.md "Programmatic
  Claude calls", all four lockdown flags are required; `AI_PROFILE.edit`
  bakes them in: tools `[Edit, Glob, Grep, Read]`, no Bash/Write/network,
  `permissionMode: acceptEdits`):

```mts
import { spawnAiAgent, pickAgent } from '@socketsecurity/lib/ai/spawn'
import { AI_PROFILE } from '@socketsecurity/lib/ai/profiles'
import { discoverAiAgents } from '@socketsecurity/lib/ai/discover'

const agent = pickAgent(await discoverAiAgents())
await spawnAiAgent(agent, {
  profile: AI_PROFILE.edit,
  prompt:
    'Re-port the behavioral changes in this upstream wrapAssembly.ts diff ' +
    'into packages/yoga-layout-builder/src/wrapper/wrapAssembly.mts, ' +
    'preserving our .mts import style and the Lock-step header. Do NOT ' +
    'touch YGEnums.mts (it is build-generated). Diff:\n' + wrapAssemblyDiff,
})
```

The build's smoke test (`generate-sync.mts` cross-checks
`YGEnums[name] === Yoga[name]` and exercises `Node.create` /
`calculateLayout`) is the feedback loop that catches a botched re-port.
**Always have a human review the re-ported diff before merge** — the
memory-ownership patches are the easy thing to get subtly wrong. Then
bump `forked_at_sha` in the `yoga-wrapassembly` row of `.config/lockstep.json`.

## Edge Cases

### ink Dependency

ink depends on yoga-sync.mjs from yoga-layout-builder. If updating both Yoga and ink, update Yoga FIRST, rebuild, then update ink.

### External Tools Dependencies

Check `packages/yoga-layout-builder/external-tools.json` after updating. New Yoga versions may require newer cmake or emscripten.

### AST Transformation Breakage

The sync wrapper uses acorn + MagicString for AST-based code transformation. If upstream changes the structure of `wrapAssembly.ts` significantly, the transformation may fail. Check `generate-sync.mts` if build produces malformed output.

### Bindings Files

The build uses official C++ bindings from cloned source (`embind.cpp`, `Node.cpp`, `Config.cpp`). If these change structure, the build may fail with missing symbols.

## Rollback Procedures

### Rollback After Commit

```bash
git reset --hard HEAD~1
```

### Rollback After Push

```bash
git revert HEAD
git push origin main
```

## Troubleshooting

### Sync Wrapper Missing Node.create()

**Cause:** wrapAssembly not applied correctly during AST transformation.

**Solution:** Check `generate-sync.mts` for transformation errors. Verify wrapAssembly.mts is up to date with upstream.

### Sync Wrapper Missing DIRECTION_LTR

**Cause:** YGEnums not inlined correctly.

**Solution:** Check that the constants object is aliased as `YGEnums` in the inlined output.

### Duplicate Export Errors

**Cause:** Multiple `export default` statements after inlining.

**Solution:** Verify AST transformation removes/converts exports properly in `generate-sync.mts`.

### ink Layout Broken After Update

**Symptom:** Terminal UI renders with wrong dimensions or alignment.

**Cause:** Yoga layout calculation behavior changed.

**Solution:**
1. Check Yoga changelog for layout behavior changes
2. Test ink package: `cd packages/ink-builder && pnpm test`
3. Rollback if layout regression confirmed

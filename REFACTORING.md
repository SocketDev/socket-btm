# DRY/KISS Refactoring Opportunities

## Critical Findings

### 1. **Duplicate Clean Scripts** (HIGH PRIORITY - DRY Violation)

**Problem:** Every package has a nearly identical `clean.mjs` script (36-82 lines each).

**Current State:**
- `packages/*/scripts/clean.mjs` - 6 files with 90% identical logic
- Each manually implements: safeDelete(BUILD_DIR), cleanCheckpoint loops
- Inconsistent logging (logger.info vs logger.step vs logger.log)

**Proposed Solution:**
```javascript
// packages/build-infra/lib/clean-builder.mjs
export async function cleanBuilder(packageName, options = {}) {
  const logger = getDefaultLogger()
  const {
    buildDir = 'build',
    distDir = options.distDir, // optional
    modes = ['prod', 'dev'],
  } = options

  logger.info(`Cleaning ${packageName}...`)

  // Delete directories
  await safeDelete(buildDir)
  if (distDir) await safeDelete(distDir)

  // Clean checkpoints
  for (const mode of modes) {
    await cleanCheckpoint(path.join(buildDir, mode), '')
  }

  logger.success('Clean complete')
}
```

**Usage:**
```javascript
// packages/onnxruntime-builder/scripts/clean.mjs (3 lines!)
import { cleanBuilder } from 'build-infra/lib/clean-builder'
await cleanBuilder('onnxruntime-builder')
```

**Impact:** ~240 lines → ~18 lines (93% reduction)

### 2. **Duplicate WASM Sync Wrapper Generation** (HIGH PRIORITY - DRY Violation)

**Problem:** Identical 100+ line logic in `onnxruntime-builder` and `yoga-layout-builder` for generating synchronous CommonJS wrappers with embedded base64 WASM.

**Current State:**
- Both packages: Read WASM → base64 encode → inject into MJS → create sync wrapper
- Same AST traversal logic (using @babel/parser + @babel/traverse)
- Same instantiateWasm override pattern
- Same base64 embedding pattern

**Proposed Solution:**
```javascript
// packages/build-infra/lib/wasm-sync-wrapper.mjs
export async function generateWasmSyncWrapper(options) {
  const { wasmFile, mjsFile, outputFile, moduleName, metadata } = options

  // Read and encode WASM
  const wasmBinary = await fs.readFile(wasmFile)
  const base64Wasm = wasmBinary.toString('base64')

  // Process MJS content
  let mjsContent = await fs.readFile(mjsFile, 'utf-8')

  // Babel AST transformation (extract existing logic)
  const ast = parse(mjsContent, { sourceType: 'module' })
  traverse(ast, { /* ... shared traversal logic ... */ })

  // Generate wrapper
  const syncWrapper = `"use strict";
// AUTO-GENERATED by ${moduleName}
// ${metadata}

const base64Wasm = "${base64Wasm}";
const wasmBinary = Uint8Array.from(atob(base64Wasm), c => c.charCodeAt(0));

${generate(ast).code}

module.exports = ${moduleName};
`

  await fs.writeFile(outputFile, syncWrapper)
}
```

**Impact:** ~200 lines → ~10 lines per package (95% reduction)

### 3. **Duplicate Build Test Patterns** (MEDIUM PRIORITY - DRY Violation)

**Problem:** All `test/build-output.test.mjs` files share 70%+ identical test structure.

**Current State:**
```javascript

  if (!existsSync(wasmDir)) {

  expect(existsSync(wasmDir)).toBe(true)

it('should have valid WASM magic number', async () => {
  const wasmPath = path.join(wasmDir, 'XXX.wasm')

    return // Skip if not built yet
```

**Proposed Solution:**
```javascript
// packages/build-infra/test/wasm-test-helpers.mjs
export function createWasmTests(packageName, wasmDir, wasmFileName) {
  return {
    testDirectory: () => {
      it('should have wasm output directory', () => {
        skipIfNotBuilt(wasmDir)

    testWasmMagic: () => {
      it(`${wasmFileName} should have valid WASM magic number`, async () => {
        const wasmPath = path.join(wasmDir, wasmFileName)
        skipIfNotBuilt(wasmPath)

        const buffer = await fs.readFile(wasmPath)
        const magic = buffer.slice(0, 4).toString('hex')
        expect(magic).toBe('0061736d')
      })
    },

    testSyncWrapper: (syncJsFileName) => {
      // ... shared test logic
    }
  }
}

function skipIfNotBuilt(path) {
  if (!existsSync(path)) {
    return // Skip if not built yet
  }
}
```

**Usage:**

**Impact:** ~300 test lines → ~50 lines per package (83% reduction)

### 4. **Inconsistent Path Resolution** (KISS Violation)

**Problem:** Each `paths.mjs` file implements similar patterns but with slight variations.

**Current State:**
```javascript
// PATTERN REPEATED in 6 files:
const __filename = fileURLToPath(import.meta.url)
const __dirname = path.dirname(__filename)
export const PACKAGE_ROOT = path.resolve(__dirname, '..')
export const BUILD_ROOT = path.join(PACKAGE_ROOT, 'build')

export function getBuildPaths(mode) {
  const buildDir = path.join(BUILD_ROOT, mode)
  const sourceDir = path.join(buildDir, 'source')
  const checkpointsDir = path.join(buildDir, 'checkpoints')
  // ... package-specific paths
}
```

**Proposed Solution:**
```javascript
// packages/build-infra/lib/path-builder.mjs
export function createPathResolver(scriptUrl) {
  const __dirname = path.dirname(fileURLToPath(scriptUrl))
  const packageRoot = path.resolve(__dirname, '..')
  const buildRoot = path.join(packageRoot, 'build')

  return {
    packageRoot,
    buildRoot,

    getSharedPaths() {
      return {
        buildDir: path.join(buildRoot, 'shared'),
        sourceDir: path.join(buildRoot, 'shared', 'source'),
        checkpointsDir: path.join(buildRoot, 'shared', 'checkpoints'),
      }
    },

    getModePaths(mode, customPaths = {}) {
      const buildDir = path.join(buildRoot, mode)
      return {
        buildDir,
        sourceDir: path.join(buildDir, 'source'),
        checkpointsDir: path.join(buildDir, 'checkpoints'),
        ...customPaths(buildDir), // Package-specific extensions
      }
    }
  }
}
```

**Usage:**
```javascript
// packages/onnxruntime-builder/scripts/paths.mjs (simplified)
import { createPathResolver } from 'build-infra/lib/path-builder'

const { packageRoot, buildRoot, getSharedPaths, getModePaths } =
  createPathResolver(import.meta.url)

export { packageRoot as PACKAGE_ROOT, buildRoot as BUILD_ROOT }
export { getSharedPaths as getSharedBuildPaths }

export function getBuildPaths(mode) {
  return getModePaths(mode, (buildDir) => ({
    wasmDir: path.join(buildDir, 'wasm'),
    outputWasmFile: path.join(buildDir, 'wasm', 'ort.wasm'),
    // ... ONNX-specific paths only
  }))
}
```

**Impact:** ~400 lines → ~150 lines across all packages (62% reduction)

### 5. **Model Builder Duplication** (DEFERRED - Separate Packages Preferred)

**Problem:** `packages/minilm-builder` and `packages/codet5-models-builder` have similar build logic patterns.

**Current State:**
- Both use similar Python script patterns (download, convert, optimize, quantize)
- Both follow same checkpoint pattern
- Both have similar error handling
- Each has model-specific configuration

**Decision:**
**Keep packages separate** - The separate package structure is preferred because:
- ✅ Clean separation of concerns - each model is self-contained
- ✅ Independent builds - can build specific models without dependencies
- ✅ Scalable - as more models are added, they get their own packages
- ✅ Flexible - can enable/disable specific model builders independently
- ✅ Avoids cluttering a single models package as more models are added

**Current Structure:**
- `packages/minilm-builder` - MiniLM-L6 models (embeddings)
- `packages/codet5-models-builder` - CodeT5 models (code analysis)
- `packages/models` - Unified orchestrator that can build both with flags

This allows flexibility: use individual packages for focused builds, or use the unified `models` package to build multiple models together.

**No consolidation needed** - The current architecture is clean and maintainable.

# Stub System Guide

How binpress selects, embeds, and uses self-extracting stubs for cross-platform binary distribution.

## Overview

binpress creates self-extracting binaries by:
1. Selecting an appropriate stub for the target platform
2. Compressing the source binary with LZFSE
3. Embedding the compressed data into the stub
4. Adding metadata and optional configuration

## Stub Selection

### Platform Detection

binpress detects the target platform from:
1. `--target` flag (explicit)
2. `--target-platform`, `--target-arch`, `--target-libc` flags
3. Source binary format (Mach-O вҶ’ macOS, ELF вҶ’ Linux, PE вҶ’ Windows)

### Selection Logic

```
Input: source binary + target specification
  вҶ“
Detect source binary format (Mach-O/ELF/PE)
  вҶ“
Determine target architecture from binary headers
  вҶ“
Select matching stub from embedded stubs
  вҶ“
Output: appropriate stub for target
```

## Stub Embedding Process

### 1. Generate Embedded Headers

```javascript
// generate-embedded-stubs.mjs
async function generateEmbeddedStubs() {
    for (const platform of platforms) {
        const stubPath = `${stubDir}/${platform}/stub`;
        const stubData = await fs.readFile(stubPath);

        // Convert to C array
        const cArray = stubData.map(b => `0x${b.toString(16).padStart(2, '0')}`);

        // Write header file
        const header = `
            static const unsigned char stub_${platform}[] = {
                ${cArray.join(', ')}
            };
            static const size_t stub_${platform}_size = ${stubData.length};
        `;

        await fs.writeFile(`embedded_stubs/${platform}.h`, header);
    }
}
```

### 2. Select Stub at Runtime

```c
// stub_selector.c
const unsigned char* get_stub_for_platform(
    uint8_t platform, uint8_t arch, uint8_t libc,
    size_t *size_out
) {
    if (platform == PLATFORM_DARWIN && arch == ARCH_ARM64) {
        *size_out = stub_darwin_arm64_size;
        return stub_darwin_arm64;
    }
    // ... other platforms
}
```

### 3. Create Compressed Binary

```c
// compress_lief.cpp
int create_compressed_binary(
    const char *source_path,
    const char *output_path,
    const unsigned char *stub,
    size_t stub_size
) {
    // 1. Write stub to output
    write(output_fd, stub, stub_size);

    // 2. Read and compress source
    uint8_t *compressed;
    size_t compressed_size;
    compress_buffer(source_data, source_size, &compressed, &compressed_size);

    // 3. Build metadata header
    write_smol_header(output_fd, compressed_size, source_size, ...);

    // 4. Write compressed data
    write(output_fd, compressed, compressed_size);

    // 5. Use LIEF to create proper binary structure
    // (add SMOL segment/section)

    return 0;
}
```

## Cross-Platform Compilation

### Target Specification

```bash

# Separate flags
```

## Stub Execution Flow

When the compressed binary runs:

```
1. Stub entry point (main)
     вҶ“
2. Find __PRESSED_DATA section (marker search or header parsing)
     вҶ“
3. Read SMOL metadata
     вҶ“
4. Extract cache key
     вҶ“
5. Check cache: ~/.socket/_dlx/<cache_key>/node
     в”ңв”Җв”Җ Exists + valid size: goto step 8
     в””в”Җв”Җ Missing/invalid: goto step 6
     вҶ“
6. Decompress to cache directory
     вҶ“
7. Write .dlx-metadata.json
     вҶ“
8. execve() cached binary with original argv
```

## Stub Size Optimization

Stubs are optimized for minimal size:

| Component | Size |
|-----------|------|
| Entry point + logic | ~4 KB |
| LZFSE decompressor | ~2 KB |
| Platform-specific code | ~2 KB |
| **Total stub size** | **~8-10 KB** |

## Error Handling

### Stub Selection Errors

```
Error: No stub available for target platform
```
- Target platform not supported
- Check available platforms with `binpress --help`

### Compression Errors

```
Error: Failed to compress binary
```
- Source binary may be corrupted
- Check disk space

### LIEF Errors

```
Error: Failed to create SMOL section
```
- Binary format incompatible
- LIEF version mismatch
